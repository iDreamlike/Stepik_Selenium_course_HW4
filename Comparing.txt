SeleniumIDE:

1. Unittest
	Практически совместим в unittest. При условии если:
	- импортировать unittest
	- в описание класса тестов передать параметр "unittest.TestCase"

2. Импортирует много лишних (неиспользуемых) библиотек.

3. Локаторы
	Негибкий подбор локаторов. После записи действий есть выбор локаторов из списка для каждого элемента, но это долго и выбор ограничен.

4. Ассерт
	Настраивается негибко. Трудно достичь желаемой точности проверки необходимых условий для ассерта.

5. Отсутствие маркировки
	Трудно управлять большим количеством таких тестов.

6. Отсутствие параметризации
	Нельзя использовать один тест с разными параметрами. Придется хранить множесто очень похожих тестов с минимальными различиями.

7. Фикстуры
	Не использует фикстуры, как это делает pytest. В итоге код теста и набора тестов длиннее.
